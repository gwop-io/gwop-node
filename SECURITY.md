# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.1.x   | Yes                |

## Reporting a Vulnerability

Gwop handles real money on mainnet. We take security seriously.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email **support@gwop.io** with:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment (what an attacker could do)
- Your suggested fix (if any)

We will acknowledge receipt within 24 hours and aim to provide a fix or mitigation within 72 hours for critical issues.

## Scope

This policy covers:

- The `@gwop/sdk` npm package
- The Gwop REST API (`api.gwop.io`)
- Webhook signature verification
- Authentication and session management

## Recognition

We appreciate responsible disclosure. Contributors who report valid vulnerabilities will be credited in our changelog (with permission).
